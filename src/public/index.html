Get started

API Endpoint

https://altschoolblogapp.herokuapp.com/

The Altschool Blog API provides fake blog data that can be consumed from any frontend application. The GET blogs and blog routes can be accessed by any user without authetication, but in order to create a blog, update a blog, and delete a blog, you need to be autheticated to perform such operations. Note that when you signup, you can update your details and delete your account. To get more information about the API click the Git Icon on the nav bar.

To use some routes in this API, you need use the signup route to register, then use the login route to get a token which needs to be placed in the authorization header next to a Bearer string before you can access the protected routes.

sign up user


# Here is a curl example
curl \
-X POST https://altschoolblogapp.herokuapp.com/auth/signup \
-J {
        "firstName": "David",
        "lastName": "Udo",
        "email": "davidudo@gmail.com",
        "password": "davidudo"
    } \

To signup a user you need to make a POST call to the following url :

QUERY PARAMETERS

Field Type Description
pageNumber Integer (optional - default: 1) page number of the list of blogs. Presently, the page number starts from 1.
state String (optional) searches for blog posts that are in either of the two states below:
draft
published
author String (optional) searches for blogs created by an author.
title Strings (optional) looks for blogs with the specified title.
tag String searches for blogs that have a specific tag name.
orderBy String (optional) sorts blogs according to specified parameters:
readCount
readTime
createdAt
updatedAt
order String (optional - default: asc) arranged the blogs in two orders:
asc
desc

Errors

The Altschool Blog API uses the following error codes:

Error Code Meaning
400 The server cannot or will not process the request due to something that is perceived to be a client error (for example, malformed request syntax or deceptive request routing).
401 Unauthorized access. This code indicates that the client request has not been completed because it lacks valid authentication credentials for the requested resource In this case you will generate a token from the /auth/login endpoint.
404 Route not found. It means the specific route a user is trying to visit is non-existent This indicates you are using a wrong endpoint.
500 Internal server error. This indicates the server encountered an unexpected condition that prevented it from fulfilling the request. Usually this error is caused from our end.
